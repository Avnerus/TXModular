1. Running the Examples

There are examples of built systems in the TX Mod Examples folder, and the folders within it. These have names indicating what they are meant to be examples of.

Use the Open File button on the top left of the main window to open the examples.

Note - Depending on the complexity of the system, it can take a few seconds for a saved system to open.

2. Stereo and mono audio signals

When you activate a channel, it checks whether the things it is trying to connect together have the right number of channels.

For example, a channel won't activate if source bus and insert modules don't have the same number of channels.

If source bus is stereo, then use a stereo insert module - these have "st" after their name, for example "flanger st" is stereo whereas "flanger" is only mono.

If you want to convert a stereo signal to a mono one, use a "stereo to mono" insert module. Or to convert a mono signal to a stereo one, use a "mono to stereo" insert module.

3. Deleting a module or channel

In the Modules & Channels Window, the System Modules box can be used to delete individual modules by pressing the -> button - this will show delete buttons next to each module button.

Also, when you press a module's button to display its details, you will see a delete button on the top line of the module's box.

When a module is deleted, any channels where it is the source of that channel, will be deleted as well.

At the top of each channel there is a "del" button to delete the channel. This will also delete any insert modules in the channel (since they are linked directly to it), but it won't delete the source module because it could be being used by other channels. If you no longer need the source module it's best to delete it separately to save CPU resources.

4. Using a midi note controller, such as a midi keyboard, wind controller or drum pads

The following source modules can be triggered by anything that can output midi notes, including a sequencer running on another computer:

Triggered Audio sources:

Filter Synth

FM Synth

Granulator

LoopPlayer (mono & stereo)

Pluck

Sample Player (mono & stereo)

Sample Player+ (mono & stereo)

Table Synth

Wave Synth

Wave Synth+

Triggered Control & Action sources:

Env 16-stage

Env Curve

Env DADSR

Note Stacker

Trigger Impulse

In all of them, you can specify:

whether they listen to midi (by setting midi listen to on or off),

which midi channels they will listen to

which midi notes they will respond to

which midi velocities they will respond to

Note Stacker is useful for example if you want to have multiple synths or sample players that respond to different midi notes or velocities.

There are also midi note and midi velocity control source modules. You can specify which midi channels they will listen to.

These will output a control signal from the midi note number and velocity of the last midi note received. These can then be used to control other modules in the system.

Action buttons on interface screens can be set to respond to specific midi notes.

Note: Widgets (such as Action buttons) are only active when they are displayed on the screen. Therefore, if you link midi controllers to widgets, and then change to a different screen, those midi controllers won't be active until they are shown on the screen again.


For more details on widgets, see the help file: 5 Building an Interface


++++++++++++++++++++++++++++++++++++++++


5. Using continuous midi controllers, such as midi sliders, knobs, or ribbon controllers or Wii controllers

There is a midi controller source module which you can set to respond to a specific midi controller number. You can specify which midi channels it will listen to.

It will output a control signal from the midi controller it is set to. This can then be used to control other modules in the system.

There is also a midi pitchbend source module. You can also specify which midi channels it will listen to.

Widgets (such as sliders) on Interface screens can be set to respond to specific midi controllers.

Note: Widgets are only active when they are displayed on the screen. Therefore, if you link midi controllers to widgets, and then change to a different screen, those midi controllers won't be active until they are shown on the screen again.


For more details on widgets, see the help file:5 Building an Interface


You can also use a Wii controller - for details, see the help file: Using Wii Controllers


++++++++++++++++++++++++++++++++++++++++


6. Note about samples and loops

When you save a system to disk, the file created will include details of all the modules within the system as well as the folder and file names of all samples and loops used by the system. However, it will not include all the samples and loops themselves. This means that if you delete or move samples or loops on your hard disk that are part of a saved system, they will not be found when you next open that system. You will then need to add them again to the system.


For more on samples and loops, see the help file: Using Samples and Loops
